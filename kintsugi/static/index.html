Type an idea → get a playable mini-app that's guaranteed to actually work.

An AI generates an interactive quiz from your prompt. Then Kintsugi runs it in a real browser, checks that every answer path reaches a valid result, and if anything is broken it makes the model repair it and re-checks — until it works or it refuses to publish. Built for the exact problem Sekai is hiring for: creation quality & reliability of AI-generated content.

Before shipping a change, Kintsugi replays the pipeline over a suite of prompts with injected faults (dead-ends, crashes, malformed output) and measures how many Jams end up valid after 0, 1, 2… automated repairs. Clears the bar → SHIP; a regression → BLOCK. This chart is a deterministic fault-injection suite — it proves the repair loop heals every known failure type and the gate blocks correctly, reproducible on any machine with no API key (kintsugi bench).

On live models the same gate runs against real output (--real). Measured on Groq's free llama-3.1-8b, roughly half the quizzes were valid and the gate BLOCKED that config — exactly its job. (The stronger free 70B tier was daily-rate-limited during testing, which is itself why the runtime is built to degrade gracefully.)

What is this? A demo runtime that generates interactive "Jams" (quizzes), then guarantees they work via real browser execution + an automatic repair loop, with a measured release gate.  ·  Everything above runs locally.  ·  github.com/Inova117/Kintsugi
